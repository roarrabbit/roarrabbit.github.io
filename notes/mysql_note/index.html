# 前言

# SQL常用命令集

  • SELECT-从数据库在提取数据
  • UPDATE-在数据库中更新数据
  • DELETE-从数据库中删除数据
  • INSERT INTO- 插入新的数据到数据库中
  • CREATE DATABASE-创建一个新的数据库
  • ALTER DATABASE-创建一个新的数据库
  • CREATE TABLE-创建新表
  • ALTER TABLE-修改表
  • DROP TABLE-删除一个表
  • CREATE INDEX-创建索引（搜索键）
  • DROP INDEX-删除索引

# 数据类型

  • INT整数 （长度若很长，则默认修改BIGINT）
  • FLOAT小数
  • CHAR字符串（保存0和1）
  • DATE日期（也是字符串，但是指定了日期）
  • TEXT可变长度的字符串
  • DECIMAL十进制（小数）10,2（表示为长度，小数位）
  • Blob是一个二进制的对象
  • 长度，长度会占系统的空间，太长则占用很大空间，太短用户使用的时候则会被限制输入长度

# 基础

  1. SQL语句是不区分大小写的
  2. 此篇的SQL固定语句都会用大写英文来表示
  3. 小写的内容一般为可改的内容
  4. 在SQL中命名不建议使用中文

    • 在创建表的时候添加外键

      foreign key (表名1)

on delete restrict	#若当要删除book_name表名2的值时，若表名1列还有该值的记录时，拒绝对book_name的表名2的删除操作
on update restrict	#若当要更新book_name表名2的值时，若表名1列还有该值的记录时，拒绝对book_name的表名2的更新操作
on delete cascade	#当要删除book_name表名2的值时，同步对book_name的表名2的删除操作

  • CREATE TABLE IF NOT EXISTS "table_name"(...)

    name CHAR(10) NOT NULL COMMENT '名字',

# 创建用户

  • 允许全部

    create user "root"@"%" identified by "password";允许root从任何IP访问（不建议）

  • 针对ip

    create user "root"@"192.168.1.10" identified by "password";允许root从192.168.1.10通过password访问

  • 指定用户通过“指定IP”访问“所有”的数据库

    grant all privileges on *.* to "root"@"192.168.1.99" identified by "password";这里指定root只能通过192.168.1.99通过password这个密码来访问所有的数据库

  • 指定用户通过“任何ip”访问“指定”的数据库

    grant all privileges on bit.* to "bit"@"%" identified by "password" with grant option;这里指定bit允许通过任何ip通过password这个密码来访问bit数据库
